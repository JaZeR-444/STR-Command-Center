# 11 Analytics & Performance - Directory Structure and Post-Launch Deferral

**CRITICAL GOVERNANCE NOTE**

The `11_Analytics_Performance` folder intentionally utilizes a **hybrid directory structure** (process/analysis documents in the root, CSV trackers in numbered subfolders). 

## Pre-Launch Deferral Strategy (Active State)
This property is currently in a **Pre-Launch State**. As such, the active folder contains the "Scaffolded" performance frameworks:
- `Analytics_Strategy.md`
- `KPI_Dashboard.md`
- `Performance_Feedback_Loop.md`
- `Pricing_Performance_Analysis.md`
- The empty baseline CSV trackers in subfolders `01` through `05`.

**DEFERRED DOCUMENTS:**
The core execution process documents (`11B` through `11K`) and the recurring execution tasks (`ANA-001`, `FBL-001`, `KPI-001`, `SEN-001`) are explicitly **deferred to post-launch**. 

Generating these analysis documents prior to receiving actual guest bookings, revenue data, and physical reviews would violate the repository's strict anti-hallucination mandate. The current scaffolded documents are sufficient to achieve "Pre-Launch Ready" status.

## Active Subfolders
1. **`01_Occupancy/`**: Contains the active occupancy tracker schema.
2. **`02_ADR_RevPAR/`**: Contains the active revenue tracker schema.

## Rule Enforcement
Do not attempt to automatically generate analysis documents or populate the CSV trackers with simulated data. The section is complete for its current pre-launch phase and will activate its continuous execution loop only after the first guest stays are completed.
